# TREND_PULLBACK_DETERMINISTIC — causal scanner-plan execution (0.2.0)

This version has no agent decision loop. On the one revealed reclaim-day bar
carrying `scanner_plan`, the policy records an `ARM_BUY_STOP` using exactly
the scanner's trigger, stop, ATR, gap budget, and expiry. Every other revealed
bar receives `OBSERVE`.

The deterministic execution engine is the sole authority for fills, gap/expiry
cancellation, stops, target1/target2, breakeven after target1, and end-of-window
flattening. It owns the scanner target ladder; no scale, cancellation, stop
alteration, or discretionary exit is permitted in this baseline.

## Setup definition (scanner; sealed into entries.db) — **0.2.0 admission**

Unchanged thesis (Lance trending pullback → reclaim EMA20), **tighter gates** vs 0.1.0:

| Gate | 0.1.0 | 0.2.0 |
|---|---|---|
| EMA tag | low within 1.5% above EMA | **low ≤ EMA** (true tag) |
| Body dip | not required | **≥1 close ≤ EMA** in pullback window |
| Pullback depth | 2–25% | **4–18%** of prior swing high |
| Pullback length | 2–12 bars | **3–10 bars** |
| Extension at reclaim | ≤8% above EMA | **≤4%** above EMA |
| Cooldown | 5 bars | **12 bars** between arms |
| SPY regime | off | **SPY close > SMA50** required |

- Uptrend: close > SMA50 and SMA200; SMA50 rising (unchanged).
- Trigger = reclaim-day high; stop under pullback low; T1 prior swing high; T2 measured move.

## Validity

Still a research baseline. No profitability claim. Promote only after a fixed
holdout with the 0.2.0 scanner cohort (not a re-label of 0.1.0 entries).
