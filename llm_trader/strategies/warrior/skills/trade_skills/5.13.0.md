# Warrior 5.13 — completed-bar decision / next-open execution

This version changes **execution timing only** from v5.12. It is not an alpha
claim and must not be tuned on historical results.

A left-labelled five-minute scanner bar at `09:35` completes after the `09:39`
one-minute bar. The deterministic policy evaluates that completed information at
the close of the final constituent minute, queues an order, and the engine may
fill it only at the following `09:40` one-minute open with configured adverse
slippage and participation limits. It never fills at the same close that supplied
the confirmation evidence.

All scanner-event, candlebar, stop, bracket, management, no-reentry, and audit
rules are unchanged from v5.12. A next-open gap at or below the structural stop
cancels the pending entry rather than manufacturing an invalid fill.

Historical runs using this version are explicitly exploratory because scanner
selection still uses a non-point-in-time float snapshot. The intended validation is
a frozen forward shadow cohort with contemporaneously captured scanner inputs.
