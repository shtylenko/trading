# TREND_PULLBACK_DETERMINISTIC — causal scanner-plan execution

This version has no agent decision loop. On the one revealed reclaim-day bar
carrying `scanner_plan`, the policy records an `ARM_BUY_STOP` using exactly
the scanner's trigger, stop, ATR, gap budget, and expiry. Every other revealed
bar receives `OBSERVE`.

The deterministic execution engine is the sole authority for fills, gap/expiry
cancellation, stops, target1/target2, breakeven after target1, and end-of-window
flattening. It owns the scanner target ladder; no scale, cancellation, stop
alteration, or discretionary exit is permitted in this baseline.

## Setup definition (scanner; already sealed into entries.db)

- Daily uptrend: close > SMA50 and SMA200; SMA50 rising.
- Pullback tags 20 EMA, then reclaim close back above EMA20.
- Trigger = reclaim-day high; stop under pullback low; T1 prior swing high;
  T2 measured move.
