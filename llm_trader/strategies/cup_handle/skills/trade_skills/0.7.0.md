# CUP_HANDLE_DETERMINISTIC — causal scanner-plan execution

This version has no agent decision loop. On the one revealed completed-handle
bar carrying `scanner_plan`, the policy records an `ARM_BUY_STOP` using exactly
the scanner's trigger, stop, ATR, gap budget, and expiry. Every other revealed
bar receives `OBSERVE`.

The deterministic execution engine is the sole authority for fills, position
size, entry-gap cancellation, expiry cancellation, stops, target1/target2,
breakeven after target1, and end-of-window flattening. It owns the scanner
target ladder; no scale, cancellation, stop alteration, or discretionary exit
is permitted in this baseline.

The harness validates the full sealed stream before it writes an intent. Audit
then re-derives every expected decision from that sealed stream and rejects a
leaf whose persisted records or policy provenance differ. This is the baseline
for evaluating later, explicitly specified deterministic cancellation or exit
predicates.
